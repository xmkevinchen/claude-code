## Context
Claude Code 的源码不只是一个工具的实现——它是 AI 编程能力发展方向的具体体现。从源码中可以看到：多 Agent 自主协作、自动规划和执行、自动代码审查、context compaction（无限对话）、proactive assistant（KAIROS）、background memory consolidation（autoDream）、IDE 深度集成等。这些能力组合起来，暗示着软件开发流程的根本性变革。

对程序员：Claude Code 已经能自主完成 plan→code→test→review→commit 全流程。当 Agent Teams 成熟后，多个 AI agent 可以并行开发一个完整 feature。这对不同层级的程序员意味着什么？哪些技能会贬值？哪些会升值？

对普通人：当 AI 能写代码、做设计、自动化工作流，软件开发的门槛和成本会大幅降低。这对就业、教育、创业、社会结构意味着什么？

## Constraints
- 分析基于源码中可观察的能力和发展方向，不是纯推测
- 需要区分"已实现的能力"和"feature-flagged 的未来能力"
- 保持客观，既不过度乐观也不过度悲观

## Key Questions
- 从源码看，Claude Code 的自动化能力已经到了什么程度？离"完全自主开发"还差多远？
- 哪些类型的编程工作最先被替代？哪些最后？
- 对非程序员来说，"人人都能编程"的时代会带来什么？
- 教育体系（CS 专业、编程培训）需要怎么调整？
- 这个趋势对社会不平等、就业结构会有什么影响？
